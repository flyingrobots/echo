Goal, method, and metric

The policy study isolates topology overhead inside execute_parallel_with_policy. Every case runs the same independent workload, so the only moving parts are shard assignment, worker shape, and delta accumulation policy.

What each case actually does

  1. Build a fresh graph store with independent benchmark nodes.
  2. Create one ExecItem per node with the same executor.
  3. Run the exact same item set under one policy at one load.
  4. Black-box the returned deltas so Criterion measures the real path.

How it is measured

  • Harness: Criterion group parallel_policy_matrix
  • Loads: 100, 1000, 10000
  • Warmup: 2s
  • Measurement window: 5s
  • Sample size: 40
  • Primary metric: mean runtime per invocation
  • Hover tooltips show exact mean and confidence interval
