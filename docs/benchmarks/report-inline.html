What this study is testing

The policy study holds the workload constant, then measures policy cost through execute_parallel_with_policy plus canonical delta merge. Only the execution shape changes: shard assignment, worker layout, and delta grouping.

What stays constant

  1. Start from a fresh graph store with independent benchmark nodes.
  2. Create one ExecItem per node with the same executor and the same item mix.
  3. Run that identical item set under one policy at one load, then canonically merge the returned deltas.
  4. Black-box the merged delta so Criterion measures the full policy path rather than dead-code-eliminating it away.

How to read the numbers

  • Harness: parallel_policy_matrix
  • Loads: 100, 1000, 10000
  • Warmup: 2s
  • Measurement window: 5s
  • Sample size: 40
  • Primary metric: mean runtime per invocation
  • Hover tooltips keep the exact mean and confidence interval on demand
