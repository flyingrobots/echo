# v0.1.0 jedit Next Ten Slices

Status: design plan for the next cross-repo release-gate batch.

This packet defines the next ten slices after the merged jedit release-gate
bootstrap. It incorporates the trusted runtime-control doctrine from
`docs/design/trusted-runtime-control-history.md`:

```text
Start/Stop are trusted runtime-control history.
Start/Stop are not application/domain intents.
Start/Stop authorize scheduler opportunities; they do not create ticks.
TickReceipt remains scheduler-owned execution evidence.
```

The goal is still the `v0.1.0` release gate:

```text
jedit-owned contract/adapters
-> Wesley generated artifacts
-> Echo generic package install
-> app-safe intent submission
-> trusted host runtime control
-> scheduler-owned tick
-> observable outcome
-> bounded QueryView reading
-> retained evidence
-> replay proof
```

## Pull Request Cadence

Open a PR after slice 5 and another after slice 10 unless a slice requires a
small supporting PR in Wesley. Echo and jedit may have paired PRs. Echo must
remain generic; jedit owns product nouns and product-facing adapter language.

without private runtime access or editor/text nouns in Echo.

## Slice 5: jedit Replay Witness Shell

Repository: `jedit`

Claim:

```text
jedit can rerun or replay enough of the Echo-backed witness to prove
deterministic reproduction shape.
```

Design work:

- Add a replay phase to the witness CLI.
- Capture the initial witness report and compare outcome/reading identity on
  replay or re-observation.
- If full durable replay is unavailable, emit a typed replay obstruction rather
  than claiming success.
- Runtime-control history should explain whether the host was started, stopped,
  drained, or left idle during the proof.

Witness:

- `node scripts/jedit-echo-witness.mjs --json --replay` or equivalent;
- same outcome/reading identity, or explicit replay obstruction;
- no app-facing tick authority.

Current implementation note: jedit can request durable replay posture with
`node scripts/jedit-echo-witness.mjs --json --replay` and currently receives
`durable_replay_unavailable` when the witness report lacks durable replay
proof. The session witness also has a local replay shell through
`node scripts/jedit-echo-powered-session.mjs --json --replay-local`, which
compares stable semantic identity and records that wall-clock cadence is not
semantic. Durable replay remains honestly unavailable until Echo/WSC recovery
closes.

## Slice 6: Echo Product-Facing Intent Outcome API Polish

Repository: `echo`

Claim:

```text
Apps can ask what happened to a submitted intent without touching trusted
runtime control.
```

Design work:

- Stabilize app-safe statuses: unknown, pending, applied, rejected, obstructed.
- Include contract package context, submission id, ticket/receipt correlation,
  and retained evidence refs where lawful.
- Keep Start/Stop/control-generation inspection read-only if surfaced at all.
- Do not expose `tick_once`, `run_until_idle`, or raw control exports through
  the app facade.

Witness:

- app-safe API observes outcomes but cannot tick;
- trusted host runtime-control policy remains separate;
- jedit can consume the outcome surface.

Current implementation note: Echo's app-facing `IntentOutcomeReceipt` now
carries installed contract context plus retained receipt evidence posture for
generated contract outcomes. Until durable receipt retention exists, the
surface reports a `MissingCoordinate` for the contract receipt coordinate
instead of fabricating a retained byte reference. Missing contract receipt
evidence also obstructs against the same retained coordinate. The app facade
still exposes only submit/observe surfaces; tick and drain authority remain on
the trusted host.

## Slice 8: Generated Package Install Path

Repositories: `echo`, `jedit`; `wesley` only if artifact descriptors are
missing.

Claim:

```text
jedit installs its generated contract package into Echo through the generic
installed-package boundary.
```

Design work:

- Echo accepts generic package metadata: package id, schema hash, artifact hash,
  codec id, supported operations, supported queries, and observer plans.
- jedit supplies generated package descriptors.
- Unsupported mutation/query ids are rejected at the package boundary.
- Host Start/Stop/cadence remains outside package install.

Witness:

- jedit witness installs the generated package;
- unsupported op/query test returns typed rejection;
- no jedit nouns enter Echo.

Current implementation note: jedit now exposes an explicit
structural-history package descriptor built from Wesley-generated
`replaceTextRange` metadata. The descriptor installs through the same generic
Echo contract-package port as the transitional hot-text runtime descriptor.
Package preflight now checks the descriptor's own required operations, so the
structural-history package can reject unsupported queries at the package
boundary without being forced to carry hot-text observer operations. The real
mutation/query round trip is still owed by slice 9.

## Slice 9: jedit Real Mutation And Query Round Trip

Repository: `jedit`

Claim:

```text
A jedit-owned edit mutation runs through Echo, then a bounded text reading
comes back with Echo evidence.
```

Design work:

- Submit `replaceTextRange` through the app-safe transport.
- Use trusted host runtime-control commands to enable/drain scheduler-owned
  execution.
- Observe applied, rejected, or obstructed outcome.
- Query a bounded text reading and return product payload plus
  `ReadingEnvelope` evidence.

Witness:

- real witness proves non-trivial vars, trusted-host runtime control, outcome
  observation, bounded reading, retained refs, and app/host authority split.

Current implementation note: the jedit powered-session witness now reports an
explicit round-trip record. The structural-history package owns the
`replaceTextRange` mutation and receipt coordinate, while the transitional
hot-text package still owns the bounded `textWindow` query observer and reading
coordinates. This keeps the release proof honest: the product mutation identity
is structural-history, the reading path remains bounded and evidence-bearing,
and the host lifecycle remains trusted-control-only. Slice 10 still owes the
non-happy-path release-gate report.

## Release Bar After Slice 10

After slice 10, the expected claim is:

```text
A real sibling app can submit a generated contract intent to Echo, without tick
authority, let a trusted host control scheduler opportunities, observe the
outcome, query a bounded reading, inspect retained evidence, and replay the
result or report a precise replay obstruction.
```

That is not the full `v0.1.0` release. It is the serious external-consumer
proof needed before final release-candidate cleanup.

## Non-Goals Across All Ten Slices

- Do not implement full Continuum transport/import.
- Do not implement full observer-rights governance.
- Do not build all of jedit.
- Do not add jedit nouns to Echo.
- Do not expose raw trusted runtime control to app code.
- Do not make Start/Stop application intents.
- Do not make wall-clock cadence semantic history.
- Do not make app dispatch synchronous with execution.
- Do not remove the fake jedit transport before the real witness is stable.
