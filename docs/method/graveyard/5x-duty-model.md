# 5x Duty Model

**Rejected.** The 5x Duty Model (documentation + implementation +
interactive demo + living test + certification from a single source)
was an ambitious vision for Echo's methodology. It was never practiced.

The concept of living specs compiled from kernel code is sound, but
the full five-way unification was aspirational overhead that blocked
progress. METHOD replaces this with a simpler loop: design, test,
build, witness, retro.

If living specs become practical, they can be proposed as a cycle
under the DOCS legend.
