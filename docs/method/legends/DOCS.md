# DOCS

Documentation.

## What it covers

Guides, specifications, living docs, course material, entry points.
Keeping what Echo says about itself honest and grounded in what the
code actually does.

## What success looks like

A newcomer can read `docs/guide/start-here.md` and understand what
Echo is, how it works, and where to look next. Every spec describes
implemented behavior, not aspirations. Stale docs are archived or
deleted, not left to mislead.

## How you know

- No spec exists for unimplemented features.
- `DOCS_AUDIT.md` tracks the state of every file.
- Guide entry points link to real, working content.
- The audit says "from scratch? yes" for every file that remains.
